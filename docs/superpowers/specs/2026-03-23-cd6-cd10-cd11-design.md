## CD.6 ‚Äî Jel Morph Efekti (Lightweight)

### Amac
Sentez aninda hucreye scale pulse + specular glow artisi ekleyerek "ozel an" hissini guclendirmek.

### Degisiklikler

**1. `SynthesisPulseCell` wrapper** (`effects/cell_effects.dart`)
- `isSynthesisResult` true oldugunda aktif
- Scale animasyonu: 1.0 ‚Üí 1.12 ‚Üí 1.0 (300ms, `Curves.easeOutBack`)
- SquashStretchCell pattern'i ile ayni lazy controller yaklasimi: nullable `_ctrl?` + `_ensureController()`
- **Disposal:** `dispose()` icinde `_ctrl?.dispose()` zorunlu ‚Äî AnimationController leak onlemi
- Reduce motion: `shouldReduceMotion` true ise scale skip, sadece glow

**2. `GelCellPainter` ‚Äî `isGlowing` parametresi**
- `isGlowing: true` ise:
  - Specular highlight alpha: 0.62 ‚Üí 0.90
  - Dis glow blur: 6 ‚Üí 12, alpha: 0.40 ‚Üí 0.60
  - Ic parlama noktasi alpha: 0.35 ‚Üí 0.60
- **`shouldRepaint`:** `old.isGlowing != isGlowing` kontrolu eklenir
- **Cache invalidation:** `_cachedGlowPaint` cache key'ine `isGlowing` dahil edilir ‚Äî glow toggle'da stale paint onlenir

**3. `CellRenderData` ‚Äî mevcut `isSynthesisResult` kullanilir**
- Yeni alan gerekmez
- GameCellWidget'ta `isGlowing: data.isSynthesisResult` ile GelCellPainter'a gecilir

### Timing
- Scale pulse (300ms): Bloom flash (0-350ms) ile esanli baslar
- Glow (600ms): `synthesisGlowTimer` ile kontrol edilir (`game_callbacks.dart`)
- Scale bittikten sonra (300ms) glow devam eder (600ms) ‚Äî kasitli davranis: hucre once "pulse" yapar, sonra yumusak glow'la soler
- Rapid cascade'de birden fazla sentez olursa: her hucre kendi `SynthesisPulseCell` instance'ina sahip, timer toplu temizler

### Reduce Motion
- `shouldReduceMotion(context)` true ise scale animasyonu skip
- Glow intensity artisi her zaman uygulanir (statik, animasyon degil)

### Kabul Kriterleri
- [ ] Sentez hucresinde 300ms scale pulse gorunur
- [ ] GelCellPainter glow intensity sentez aninda artar
- [ ] `_cachedGlowPaint` isGlowing degisiminde invalidate olur
- [ ] `SynthesisPulseCell.dispose()` controller'i temizler ‚Äî leak yok
- [ ] Reduce motion'da scale skip edilir
- [ ] SquashStretchCell ve WaveRipple animasyonlari hala dogru calisir (regresyon)
- [ ] Mevcut testler geciyor

## CD.10 ‚Äî Gorev Sistemi Genisletme

### Amac
Gunluk gorev havuzunu 12+ goreve cikarmak, haftalik gorevleri aktif etmek.

### Degisiklikler

**1. Gunluk havuz genisletme** (`quests.dart`)
Mevcut 6 goreve ek varyasyonlar. **Onemli:** Ayni QuestType farkli hedeflerle havuzda olabilir. Progress key carpismasini onlemek icin her Quest'e unique `id` alani eklenir.

| # | id | QuestType | Hedef | XP | Jel Ozu |
|---|-----|-----------|-------|----|---------|
| 7 | d_cl15 | clearLines | 15 | 40 | 3 |
| 8 | d_rs500 | reachScore | 500 | 35 | 2 |
| 9 | d_ms8 | makeSyntheses | 8 | 50 | 4 |
| 10 | d_rc5 | reachCombo | 5 | 60 | 4 |
| 11 | d_pg5 | playGames | 5 | 45 | 3 |
| 12 | d_cs10 | clearLines | 20 | 55 | 4 |

> **Not (Review CR-I2):** `useColorSynthesis` QuestType semantik olarak `makeSyntheses` ile ayni ‚Äî kaldirilir, havuza eklenmez. Yerine farkli hedefli `clearLines` varyasyonu konur.

**2. Progress key degisikligi** (`quest_provider.dart`)
- Mevcut key format: `'${type.name}_d'` ‚Üí **yeni:** `'${quest.id}'`
- Her Quest'in unique `id`'si progress key olarak kullanilir
- Ayni QuestType'tan 2 gorev secilse bile progress bagimsiz takip edilir
- **Sinif adi:** `QuestProgress` (mevcut ad, spec'teki `QuestState` hatasi duzeltildi)

**3. Haftalik progress tracking** (`quest_provider.dart`)
- `incrementQuestProgress` ‚Üí gunluk (`d_*`) VE haftalik (`w_*`) anahtarlarini paralel guncelle
- `QuestProgress`'e `weeklyQuests: List<Quest>` alani ekle
- **Haftalik seed:** `DateTime.now().year * 100 + isoWeekNumber` ‚Äî takvim haftasiyla uyumlu (Pazartesi-Pazar)
- Haftalik gorev secimi: `kWeeklyQuestPool`'dan 5 gorev

**4. Haftalik persistence** (`local_repository.dart`)
- `getWeeklyQuestProgress()` / `saveWeeklyQuestProgress(Map<String, int>)`
- `getWeeklyQuestWeek()` / `saveWeeklyQuestWeek(String)` ‚Äî ISO hafta key
- Hafta degistiginde `w_*` progress sifirlanir

**5. QuestBar guncelleme** (`quest_bar.dart`)
- Haftalik gorev progress bar'i (kOrange accent, gunluk kGold'un altinda)
- Veya: mevcut QuestBar'da gunluk/haftalik toggle

### Haftalik Reset Mekanizmasi
- `quest_provider.dart` init'te ISO hafta key kontrolu
- Hafta degistiyse `w_*` progress temizlenir

### Kabul Kriterleri
- [ ] Gunluk havuz 12+ gorev
- [ ] Ayni QuestType'tan 2 gorev secilse bile progress bagimsiz
- [ ] Haftalik gorevler tracking ve persist calisiyor
- [ ] Haftalik progress app restart sonrasi korunuyor
- [ ] Hafta degisiminde progress sifirlanir (ISO hafta bazli)
- [ ] Provider guncellenmesi UI'a yansiyor
- [ ] Mevcut quest testleri geciyor

## CD.11 ‚Äî Push Notification Entegrasyonu

### Amac
Stub NotificationService'i Firebase Cloud Messaging + yerel bildirimlerle degistirmek.

### Bagimlilklar
```yaml
firebase_messaging: ^15.1.0
flutter_local_notifications: ^18.0.0
timezone: ^0.9.4
```

### Degisiklikler

**1. `NotificationService` interface genisletme** (`notification_service.dart`)
- `Future<void> initialize()` metodu eklenir
- `StubNotificationService.initialize()` ‚Üí no-op (mevcut stub derlenir)

**2. `FirebaseNotificationService`** (`notification_service.dart`)
- `NotificationService` interface'ini implemente eder
- `initialize()`:
  - `tz.initializeTimeZones()` + `tz.setLocalLocation(tz.local)` ‚Äî **timezone init zorunlu** (flutter_local_notifications `zonedSchedule` icin gerekli)
  - `FlutterLocalNotificationsPlugin().initialize()` ‚Äî kanal ayarlari
  - `FirebaseMessaging.instance.requestPermission()` ‚Äî FCM izin
- `requestPermission()`: FCM + local notification izinleri. **Permission denied ‚Üí graceful no-op**, retry spam yok
- `getToken()`: FCM token doner, Supabase'e sync
- `scheduleStreakReminder()`: Her gun 20:00 local notification (`zonedSchedule`)
- `scheduleDailyPuzzleReminder()`: Her gun 10:00 local notification (`zonedSchedule`)
- `scheduleComebackNotification()`: 3 gun sonrasi local notification
- `cancelNotification(id)` / `cancelAll()`: Iptal
- **Token refresh:** `FirebaseMessaging.instance.onTokenRefresh.listen()` ‚Äî subscription `dispose()`'da cancel edilir

**3. Notification string'leri**
- `scheduleStreakReminder(title:, body:)` string parametresi alir
- String'ler **zamanlama aninda** l10n'den resolve edilir (fire aninda app kapali olabilir)
- Cagiran taraf `ref.read(stringsProvider)` ile lokalize string gecer

**4. Platform yapilandirmasi**
- **Android:** `POST_NOTIFICATIONS` (Android 13+) + `RECEIVE_BOOT_COMPLETED` + `WAKE_LOCK` izinleri
- **iOS:** `UIBackgroundModes: [remote-notification]` + `Runner.entitlements` push capability
- **Web:** Desteklenmez (`kIsWeb` guard)

**5. Provider** (`service_providers.dart`)
- `notificationServiceProvider` ‚Üí `FirebaseNotificationService()` (web'de `StubNotificationService`)

**6. Init** (`main.dart`)
- `kIsWeb` guard ile `await notificationService.initialize()`
- `WidgetsBindingObserver.didChangeAppLifecycleState` hook:
  - **Foreground resume:** comeback notification iptal et + yeniden zamanla
  - **Background:** comeback notification aktif

**7. Scheduler entegrasyonu**
- `HomeScreen.initState`: streak reminder + daily puzzle zamanlama
- `game_callbacks.dart` game over: comeback notification guncelle (3 gun sonrasi)
- `Settings`: Notification toggle (enable/disable) ‚Äî disable ise `cancelAll()`

**8. FCM Token Sync** (`remote_repository.dart`)
- `saveDeviceToken(String token)` ‚Üí Supabase `device_tokens` tablosu
- **Migration gerekli:** `supabase/migrations/YYYYMMDD_device_tokens.sql`
  ```sql
  CREATE TABLE device_tokens (
    id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
    token text NOT NULL,
    platform text NOT NULL,
    updated_at timestamptz DEFAULT now()
  );
  ALTER TABLE device_tokens ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Users manage own tokens" ON device_tokens
    FOR ALL USING (auth.uid() = user_id);
  CREATE UNIQUE INDEX idx_device_tokens_user_platform
    ON device_tokens(user_id, platform);
  ```
- Token yenilendiginde `onTokenRefresh` listener ile otomatik guncelleme

### Harici Gereksinimler (Manuel)

- iOS: APNs sertifikasi (p8 key) Firebase'e yuklenmeli
- Android: FCM server key otomatik (google-services.json mevcut)
